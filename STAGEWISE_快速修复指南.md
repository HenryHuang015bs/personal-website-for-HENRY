# Stagewise 快速修复指南

## 当前状态

✅ 扩展已安装：`stagewise.stagewise-vscode-extension-0.11.3-universal`  
✅ 配置文件存在：`stagewise.json` (端口 3000)  
✅ 组件已集成：`StagewiseInit` 在 `layout.tsx` 中  
❌ 扩展未在页面中检测到

## 快速修复步骤

### 步骤 1：重新加载 Cursor 窗口（最重要）

1. **在 Cursor 中按 `Cmd+Shift+P` (Mac) 或 `Ctrl+Shift+P` (Windows)**
2. **输入并选择：`Developer: Reload Window`**
3. **等待窗口重新加载完成**
4. **刷新浏览器页面** (`Cmd+R` 或 `F5`)

### 步骤 2：验证扩展是否激活

1. **在 Cursor 中按 `Cmd+Shift+X` 打开扩展面板**
2. **搜索 "Stagewise"**
3. **确认扩展显示为"已启用"（Enabled）**
4. **如果显示"禁用"，点击启用**

### 步骤 3：检查开发服务器

确保开发服务器正在运行：
```bash
npm run dev
```

确认服务器运行在端口 3000（检查终端输出）

### 步骤 4：验证浏览器连接

1. **打开浏览器访问：`http://localhost:3000`**
2. **打开浏览器开发者工具**（F12 或 Cmd+Option+I）
3. **查看控制台**，应该看到诊断信息
4. **等待 30-60 秒**，看是否有扩展连接消息

### 步骤 5：尝试点击元素

如果扩展已连接但工具栏未显示：
- 尝试点击页面上的任意元素
- 尝试双击元素
- 尝试右键点击元素
- Stagewise 工具栏可能在第一次交互时出现

### 步骤 6：完全重启（如果以上步骤无效）

1. **完全退出 Cursor**（不是关闭窗口，而是退出应用）
   - Mac: `Cmd+Q`
   - Windows: 关闭所有窗口并退出
2. **等待 5 秒**
3. **重新打开 Cursor**
4. **打开项目**
5. **等待几秒让扩展激活**
6. **刷新浏览器页面**

## 验证扩展连接的方法

### 方法 1：检查浏览器控制台

打开浏览器控制台，查看是否有：
- `✅ 发现 window.__STAGEWISE__`
- `✅ Stagewise 扩展已连接！`
- `✅ 检测到 Stagewise 元素被注入!`

### 方法 2：检查 Cursor 状态栏

查看 Cursor 编辑器右下角是否有 Stagewise 相关的状态指示

### 方法 3：检查命令面板

1. 在 Cursor 中按 `Cmd+Shift+P`
2. 输入：`Stagewise` 或 `set agent`
3. 如果看到 `Stagewise: Set the preferred agent...`，说明扩展已激活

## 常见问题

### Q: 扩展已安装但控制台显示未检测到？

**A:** 扩展可能需要时间连接到浏览器。尝试：
1. 等待 30-60 秒
2. 重新加载 Cursor 窗口
3. 刷新浏览器页面

### Q: 扩展显示已启用但没有反应？

**A:** 可能是扩展未正确激活。尝试：
1. 禁用扩展
2. 重新启用扩展
3. 重新加载窗口

### Q: 控制台显示所有检查都是 ❌？

**A:** 扩展可能使用不同的连接方式。尝试：
1. 检查 Cursor 编辑器日志（帮助 > 切换开发者工具）
2. 查看 Network 标签是否有失败的请求
3. 确认 stagewise.json 配置正确

### Q: 扩展需要手动初始化吗？

**A:** 不需要。Stagewise 扩展应该自动注入到页面。如果未检测到，可能是：
- 扩展未激活
- Cursor 窗口需要重新加载
- 浏览器需要刷新

## 下一步

如果按照以上步骤操作后仍然无法检测到扩展，请：

1. **检查 Cursor 编辑器日志**
   - 在 Cursor 中：帮助 > 切换开发者工具
   - 查看控制台是否有错误

2. **检查浏览器 Network 标签**
   - 查看是否有与 stagewise 相关的请求
   - 查看是否有连接错误

3. **尝试更新扩展**
   - 在扩展面板中检查是否有更新
   - 如果有，更新到最新版本

4. **重新安装扩展**
   - 在扩展面板中卸载 Stagewise
   - 重新搜索并安装

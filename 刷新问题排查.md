# 🔧 刷新不出来 - 问题排查指南

## ✅ 当前状态
- ✅ 开发服务器正在运行（返回 200 状态码）
- ✅ 配置文件已更新（移除了可能导致问题的 webpack devtool 配置）
- ✅ `.next` 缓存已清理并重新构建

## 🔍 排查步骤

### 1. 硬刷新浏览器缓存
这些是最常见的解决方案，请按顺序尝试：

**方法 1：硬刷新**
- **Mac**: 按 `Cmd + Shift + R` 或 `Cmd + Option + R`
- **Windows/Linux**: 按 `Ctrl + Shift + R` 或 `Ctrl + F5`

**方法 2：清除浏览器缓存**
1. 打开开发者工具（F12 或 Cmd+Option+I）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

**方法 3：使用无痕模式**
- 打开浏览器无痕/隐私窗口
- 访问 `http://localhost:3000`

### 2. 检查控制台错误
打开浏览器开发者工具（F12），查看：

**Console 标签页**
- 是否有红色错误信息？
- 如果有错误，请复制错误信息

**Network 标签页**
- 刷新页面后查看哪些资源加载失败（红色）
- 检查 CSS、JS 文件是否正常加载

### 3. 检查页面内容
如果页面是空白的，请检查：
- 查看页面源代码（右键 → 查看源代码）
- 检查是否有 HTML 内容
- 检查 `<body>` 标签内是否有内容

### 4. 重启开发服务器
如果以上方法都不行，请：

1. **停止开发服务器**
   - 在运行 `npm run dev` 的终端窗口按 `Ctrl + C`

2. **清理并重新启动**
   ```bash
   rm -rf .next
   npm run dev
   ```

3. **等待服务器完全启动**
   - 看到 "Ready" 或 "Local: http://localhost:3000" 消息后再访问

## 🎯 常见问题

### 问题 1: 页面空白但服务器正常运行
**可能原因**: 浏览器缓存了旧版本
**解决方案**: 使用硬刷新（Cmd+Shift+R）或清除缓存

### 问题 2: 样式没有加载
**可能原因**: CSS 文件路径问题或 Tailwind 未编译
**检查**: 
- Network 标签页查看 `globals.css` 是否加载成功
- 检查控制台是否有 CSS 相关错误

### 问题 3: 内容显示但不完整
**可能原因**: JavaScript 错误阻止了渲染
**检查**: 
- Console 标签页查看错误
- 检查 Three.js 或其他库是否正确加载

### 问题 4: Three.js 场景不显示
**可能原因**: WebGL 不支持或图片加载失败
**检查**: 
- 控制台是否有 WebGL 相关错误
- Network 标签页检查图片资源是否加载成功

## 🔄 快速重置步骤

如果所有方法都不行，可以尝试完全重置：

```bash
# 1. 停止开发服务器（Ctrl+C）

# 2. 清理所有缓存
rm -rf .next
rm -rf node_modules/.cache

# 3. 重新安装依赖（如果需要）
# npm install

# 4. 重新启动
npm run dev
```

然后：
- 打开无痕窗口
- 访问 http://localhost:3000
- 硬刷新（Cmd+Shift+R）

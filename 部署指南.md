# 网站部署指南

## 推荐方式：Vercel 部署（最简单）

### 方法一：通过 Vercel 网站部署（推荐）

1. **准备工作**
   - 确保代码已推送到 GitHub 仓库
   - 如果没有 GitHub 仓库，先初始化：
     ```bash
     git init
     git add .
     git commit -m "Initial commit"
     git branch -M main
     ```

2. **部署步骤**
   - 访问 [https://vercel.com](https://vercel.com)
   - 使用 GitHub 账号登录
   - 点击 "Add New Project"
   - 选择你的 GitHub 仓库
   - Vercel 会自动检测到 Next.js 项目
   - 点击 "Deploy" 即可
   - 部署完成后会获得一个免费域名（如：`your-project.vercel.app`）

3. **环境变量（如果有）**
   - 如果项目需要环境变量，在 Vercel 项目设置中添加

### 方法二：使用 Vercel CLI 部署

1. **安装 Vercel CLI**
   ```bash
   npm i -g vercel
   ```

2. **登录 Vercel**
   ```bash
   vercel login
   ```

3. **部署项目**
   ```bash
   vercel
   ```
   首次部署会询问一些问题，按提示操作即可。

4. **生产环境部署**
   ```bash
   vercel --prod
   ```

### 方法三：通过 GitHub Actions 自动部署

1. 在仓库根目录创建 `.github/workflows/deploy.yml`：
   ```yaml
   name: Deploy to Vercel
   on:
     push:
       branches: [main]
   jobs:
     deploy:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v3
         - uses: actions/setup-node@v3
           with:
             node-version: '18'
         - run: npm install
         - run: npm run build
   ```

## 其他部署方式

### Netlify 部署

1. 访问 [https://netlify.com](https://netlify.com)
2. 使用 GitHub 登录
3. 选择 "Add new site" > "Import an existing project"
4. 选择你的仓库
5. 构建设置：
   - Build command: `npm run build`
   - Publish directory: `.next`

### 自托管部署（VPS/服务器）

1. **构建项目**
   ```bash
   npm run build
   ```

2. **启动生产服务器**
   ```bash
   npm start
   ```

3. **使用 PM2 管理进程（推荐）**
   ```bash
   npm install -g pm2
   pm2 start npm --name "personal-website" -- start
   pm2 save
   pm2 startup
   ```

4. **配置 Nginx 反向代理**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       
       location / {
           proxy_pass http://localhost:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```

## 部署前检查清单

- [ ] 确保所有依赖已安装：`npm install`
- [ ] 本地测试构建：`npm run build`
- [ ] 本地测试生产环境：`npm start`
- [ ] 检查环境变量配置
- [ ] 确保所有图片和静态资源路径正确
- [ ] 测试网站功能是否正常

## 常见问题

### 构建失败
- 检查 `package.json` 中的依赖是否正确
- 查看构建日志中的错误信息
- 确保 Node.js 版本兼容（建议使用 Node.js 18+）

### 图片无法显示
- 检查 `public` 文件夹中的图片路径
- 确认 `next.config.mjs` 中的图片配置

### 404 错误
- 检查路由配置
- 确认所有页面文件都在正确位置

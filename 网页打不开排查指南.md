# 网页打不开排查指南

## ✅ 当前状态

- ✅ 开发服务器正在运行（端口 3000）
- ✅ 服务器可以正常响应请求（HTTP 200）
- ✅ 项目构建成功，无编译错误
- ✅ `runtime.connect()` 错误已修复

## 🔍 排查步骤

### 1. 确认服务器正在运行

在终端运行：
```bash
npm run dev
```

应该看到：
```
▲ Next.js 14.2.25
- Local:        http://localhost:3000
```

### 2. 检查访问地址

**正确的地址：**
- ✅ `http://localhost:3000`
- ✅ `http://127.0.0.1:3000`

**错误的地址：**
- ❌ `file:///Users/...` （不要直接打开文件）
- ❌ `https://localhost:3000` （开发服务器不支持 HTTPS）

### 3. 清除浏览器缓存

**方法 1：硬刷新**
- Mac: `Cmd + Shift + R`
- Windows/Linux: `Ctrl + Shift + R`

**方法 2：清除缓存**
1. 打开开发者工具（F12 或 Cmd+Option+I）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

**方法 3：无痕模式测试**
- Mac: `Cmd + Shift + N`
- Windows/Linux: `Ctrl + Shift + N`
- 在无痕模式下访问 `http://localhost:3000`

### 4. 检查浏览器控制台

1. 打开开发者工具（F12 或 Cmd+Option+I）
2. 切换到 **Console** 标签页
3. 查看是否有红色错误信息

**常见的错误类型：**
- JavaScript 运行时错误
- 网络请求失败（404、500 等）
- CORS 错误

### 5. 检查网络请求

1. 在开发者工具中切换到 **Network** 标签页
2. 刷新页面（F5）
3. 查看是否有失败的请求（红色标记）

**需要检查的关键资源：**
- HTML 文档（应该是 200）
- JavaScript 文件（/_next/static/chunks/...）
- CSS 文件（/_next/static/css/...）

### 6. 检查终端输出

查看运行 `npm run dev` 的终端窗口，检查是否有：
- 编译错误
- 运行时错误
- 警告信息

### 7. 尝试重启服务器

1. 在终端按 `Ctrl + C` 停止服务器
2. 等待几秒钟
3. 重新运行 `npm run dev`

### 8. 检查端口占用

如果端口 3000 被占用，尝试使用其他端口：

```bash
npm run dev -- -p 3001
```

然后访问 `http://localhost:3001`

## 🚨 常见问题

### 问题 1：页面显示空白

**可能原因：**
- JavaScript 错误导致页面渲染失败
- CSS 加载失败
- 浏览器兼容性问题

**解决方案：**
1. 检查浏览器控制台的错误信息
2. 尝试在无痕模式下打开
3. 尝试不同的浏览器（Chrome、Firefox、Safari）

### 问题 2：连接被拒绝

**错误信息：** `ERR_CONNECTION_REFUSED` 或 `无法访问此网站`

**可能原因：**
- 开发服务器未运行
- 端口被其他程序占用
- 防火墙阻止连接

**解决方案：**
1. 确认 `npm run dev` 正在运行
2. 检查是否有其他程序使用端口 3000：
   ```bash
   lsof -ti:3000
   ```
3. 如果有其他程序占用，停止它或使用其他端口

### 问题 3：页面加载很慢或超时

**可能原因：**
- 网络问题
- 服务器响应慢
- 资源文件过大

**解决方案：**
1. 检查网络连接
2. 查看终端中是否有构建信息
3. 尝试清除 `.next` 缓存：
   ```bash
   rm -rf .next
   npm run dev
   ```

### 问题 4：某些功能不工作

**可能原因：**
- JavaScript 运行时错误
- API 请求失败
- 依赖未正确安装

**解决方案：**
1. 检查浏览器控制台的错误
2. 重新安装依赖：
   ```bash
   rm -rf node_modules
   npm install
   npm run dev
   ```
